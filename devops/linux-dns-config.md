# Linux DNS 服务器配置

## 核心概念
Linux 系统的 DNS 解析由 `/etc/resolv.conf` 文件配置，但现代发行版通常通过 `systemd-resolved`、`NetworkManager` 或 `netplan` 等工具管理 DNS，直接修改 `resolv.conf` 可能会被覆盖。

## 常用公共 DNS 服务器

| DNS 服务商 | 主 DNS | 备 DNS | 特点 |
|-----------|--------|--------|------|
| 阿里 DNS | 223.5.5.5 | 223.6.6.6 | 国内速度快 |
| 腾讯 DNS | 119.29.29.29 | — | 国内速度快 |
| 百度 DNS | 180.76.76.76 | — | 国内速度快 |
| 114 DNS | 114.114.114.114 | 114.114.115.115 | 国内稳定 |
| Google DNS | 8.8.8.8 | 8.8.4.4 | 全球通用但国内可能慢 |
| Cloudflare | 1.1.1.1 | 1.0.0.1 | 隐私友好 |

## 快速判断系统使用哪种 DNS 管理方式

```bash
# 1. 看 /etc/resolv.conf 是否有被管理的标记
head -3 /etc/resolv.conf

# 2. 检查是否用了 systemd-resolved
systemctl is-active systemd-resolved

# 3. 检查是否用了 NetworkManager
systemctl is-active NetworkManager

# 4. 检查是否用了 netplan
ls /etc/netplan/ 2>/dev/null

# 5. 查看运行中的 DNS 配置
resolvectl status 2>/dev/null || nmcli device show 2>/dev/null | grep DNS
```

## 方法一：直接修改 /etc/resolv.conf（通用）

```bash
sudo vim /etc/resolv.conf
```

格式：
```
nameserver 8.8.8.8
nameserver 114.114.114.114
```

⚠️ 如果文件头写着 `# Generated by ...`，说明会被自动管理工具覆盖。

## 方法二：systemd-resolved（Ubuntu 16.04+ / CentOS 7+）

**临时修改：**
```bash
sudo resolvectl dns eth0 8.8.8.8 114.114.114.114
sudo systemctl restart systemd-resolved
```

**永久修改（配置文件）：**
```bash
sudo vim /etc/systemd/resolved.conf
```

配置内容：
```
[Resolve]
DNS=8.8.8.8 114.114.114.114
FallbackDNS=1.1.1.1
```

重启服务：
```bash
sudo systemctl restart systemd-resolved
```

**配置软链接**（让传统工具也使用 systemd-resolved）：
```bash
sudo ln -sf /run/systemd/resolve/stub-resolv.conf /etc/resolv.conf
```

## 方法三：NetworkManager（CentOS/RHEL/Fedora 常用）

```bash
# 查看当前连接
nmcli connection show

# 设置 DNS
nmcli connection modify "eth0" ipv4.dns "8.8.8.8 114.114.114.114"
nmcli connection modify "eth0" ipv4.ignore-auto-dns yes

# 重新激活连接
nmcli connection down "eth0" && nmcli connection up "eth0"

# 或使用交互界面
nmtui
```

## 方法四：netplan（Ubuntu 18.04+）

```bash
sudo vim /etc/netplan/01-netcfg.yaml
```

示例配置：
```yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: true
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]
```

应用配置：
```bash
sudo netplan apply
```

## 验证 DNS 是否生效

```bash
cat /etc/resolv.conf
nslookup www.baidu.com
dig www.baidu.com
host www.baidu.com
```

## 注意事项
- 修改 DNS 后域名解析立即生效，无需重启服务器
- 国外服务器访问国内网站慢 → 改用国内 DNS
- 国内服务器访问海外服务慢 → 改用 Google/Cloudflare DNS
- 不同发行版的管理工具互不兼容，先确认再用对应方法
